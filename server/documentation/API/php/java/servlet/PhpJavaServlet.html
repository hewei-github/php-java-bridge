public class PhpJavaServlet

Handles requests from PHP clients. When apache or IIS are not available, this servlet can start php as a CGI sub-process.
However, it is recommended to install php as an Apache module and to share the servlet engine's webapps directory with apache's htdocs directory. Since Fedora Linux version 4 the tomcat webapps directory and the Apache htdocs directory are shared by default (via a symlink).
For frameworks, such as Java Server Faces, it is recommended to forward requests to the backend as follows:
<?php
function toString() {"return "hello java world!";}
java_context()->call(java_closure()) ||header("Location: hello.jsp");
?>

This allows the framework (jsf, struts, ...) to call php procedures as follows:
PhpScriptEngine e = new PhpScriptEngine();
e.eval(new URLReader(new URL("/hello.php")));
out.println(((Invocable)e).call("java_get_server_name", new Object[]{}));
e.release();
An alternative would be to install mod_jk to "JkMount" the java "webapps" document root into the "htdocs" document root of the HTTP server and to configure it so that it automatically forwards all .jsp and servlet requests to the servlet engine.
